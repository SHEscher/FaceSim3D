## Description of data

### Stimuli: 3D head models

3D-reconstructed head stimuli for the triplet-odd-one-out paradigm in the 2D and 3D viewing conditions were generated
from 2D-portraits of the [Chicago Face Database (CFD)](https://www.chicagofaces.org)
using the `DECA` algorithm by [Feng et al. (2021)](https://doi.org/10.1145/3450626.3459936).

A table of the used face images (n=100) – provided by the `CFD` – can be found in `headnmap.csv`
in `experiment/FaceSimExp/Assets/Faces/CFD/`.

Naming information and additional face features, including physical face attributes (PFA),
can be found in `data/faces/CFD/CFD 3.0 Norming Data and Codebook.xlsx`.

Due to license constraints, 3D head stimuli need to be regenerated by interested third parties.
For details, see our corresponding publication.

### Human behavioral data: face similarity judgments

Behavioral response data of the main study are shared via the corresponding public [OSF project](https://osf.io/4a2jf/ "see data/main.zip")
and [Zenodo](https://doi.org/10.5281/zenodo.19236583) (data archive).

### Embedding models

Also, model weights and data are shared in the above-mentioned [OSF project](https://osf.io/4a2jf/ "see data/models/")
and [Zenodo](https://doi.org/10.5281/zenodo.19236583) (data archive).
Original weights of the VGG-Face model (Parkhi et al., 2015) can also be loaded with the script:
`code/facesim3d/modeling/VGG/convert_weights.py`.

*Note: place all data shared via OSF and [Zenodo](https://doi.org/10.5281/zenodo.19236583) in this `data/` folder.
Keep the tree structure intact to minimize friction during data loading using the scripts in `code/`.*
